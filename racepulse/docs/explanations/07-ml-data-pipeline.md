# 07. 머신러닝 서버와 데이터 파이프라인

ML 서버는 RacePulse에서 데이터 수집, 예측, 시뮬레이션을 담당합니다.

## 사용 기술

| 기술 | 역할 |
| --- | --- |
| FastAPI | Python API 서버 |
| SQLAlchemy Async | 비동기 DB 접근 |
| asyncpg | PostgreSQL 비동기 드라이버 |
| pandas | 표 형태 데이터 처리 |
| scikit-learn | 기본 머신러닝 모델 |
| XGBoost / LightGBM | 성능 좋은 트리 기반 모델 |
| numpy | 수치 계산 |
| joblib | 학습된 모델 저장/불러오기 |
| httpx | 외부 API 호출 |
| BeautifulSoup | HTML 파싱 |

## 데이터 파이프라인이란

데이터 파이프라인은 데이터를 가져와서, 정리하고, 저장하고, 모델에 사용할 수 있게 만드는 흐름입니다.

RacePulse에서는 다음 흐름입니다.

```text
외부 KRA/날씨 API
  ↓
FastAPI 수집 서비스
  ↓
정제와 변환
  ↓
PostgreSQL 저장
  ↓
특징 생성
  ↓
ML 모델 예측
  ↓
예측 결과 저장
  ↓
Spring Boot API와 프론트엔드에서 조회
```

## KRA API 수집

KRA API는 경주 일정, 출전 정보, 결과 같은 원천 데이터를 제공합니다.

외부 API는 다음 문제가 있을 수 있습니다.

- 응답이 느림
- 호출 제한이 있음
- 특정 날짜 데이터가 비어 있음
- 필드 형식이 예상과 다름
- 네트워크 오류가 발생함

그래서 수집 코드는 재시도, 빈 값 처리, 호출 제한 관리를 고려해야 합니다.

## 날씨 API 수집

날씨는 경주 결과에 영향을 줄 수 있는 외부 변수입니다.

예를 들어 다음 정보가 예측 특징이 될 수 있습니다.

- 기온
- 강수 여부
- 습도
- 풍속
- 트랙 상태

## 특징 생성이란

특징은 ML 모델에 넣을 입력값입니다.

원본 데이터가 다음과 같다고 가정합니다.

```text
말 이름, 기수, 부담중량, 최근 성적, 경주 거리, 날씨
```

모델은 문자열 그대로를 잘 이해하지 못합니다. 그래서 숫자 또는 규칙적인 값으로 바꿔야 합니다.

예시는 다음과 같습니다.

```text
최근 3경기 평균 순위
최근 3경기 평균 기록
해당 거리 경험 횟수
기수 최근 승률
부담중량 변화량
```

이렇게 모델이 학습하기 좋은 값으로 바꾸는 과정을 특징 생성이라고 합니다.

## 모델 학습과 예측

모델 학습은 과거 데이터를 보고 패턴을 배우는 과정입니다.

예측은 새 경주 데이터에 학습된 모델을 적용하는 과정입니다.

```text
과거 데이터 + 실제 결과 → 모델 학습
새 경주 데이터 → 모델 예측 → 승률/순위
```

## 왜 XGBoost나 LightGBM을 쓰는가

경마 데이터는 숫자, 범주, 결측값, 비선형 관계가 섞여 있습니다. XGBoost와 LightGBM은 이런 표 형태 데이터에서 강한 성능을 내는 경우가 많습니다.

장점은 다음과 같습니다.

- 표 형태 데이터에 강함
- 변수 중요도를 확인하기 쉬움
- 결측값 처리에 비교적 강함
- 작은 데이터에서도 기준 모델로 쓰기 좋음

## 예측값과 몬테카를로의 관계

ML 모델은 말별 승률이나 예상 순위를 만듭니다. 몬테카를로는 이 예측값을 입력으로 사용합니다.

```text
ML 모델 예측값
  ↓
말별 승률
  ↓
몬테카를로 10,000회 반복
  ↓
순위 분포와 이변 가능성
```

즉, 몬테카를로는 모델을 대체하지 않습니다. 모델 결과를 더 해석하기 쉽게 만드는 후처리 단계입니다.

## 비동기 처리가 필요한 이유

FastAPI와 SQLAlchemy Async는 동시에 여러 I/O 작업을 처리하기 좋습니다.

외부 API 호출과 DB 저장은 CPU 계산보다 기다리는 시간이 많습니다. 비동기를 사용하면 기다리는 동안 다른 요청을 처리할 수 있습니다.

## 운영 체크리스트

1. 외부 API 키가 환경변수로 주입되는지 확인합니다.
2. 수집 실패 시 로그에 날짜, 경마장, API 이름이 남는지 확인합니다.
3. 중복 수집이 DB 제약이나 Redis 락으로 방지되는지 확인합니다.
4. 모델 파일이 없는 경우 기본 동작이 정해져 있는지 확인합니다.
5. 예측 결과가 없을 때 프론트엔드가 한글 오류 메시지를 보여주는지 확인합니다.
6. 한글 이름과 경주명이 DB와 JSON 응답에서 깨지지 않는지 확인합니다.

## 초보자가 기억할 핵심

ML 서버는 단순히 "AI를 돌리는 서버"가 아닙니다. 좋은 예측을 위해 데이터를 안정적으로 수집하고, 정리하고, 저장하고, 모델이 이해할 수 있는 형태로 바꾸는 전체 흐름을 담당합니다.
