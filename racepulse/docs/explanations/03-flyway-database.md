# 03. Flyway와 데이터베이스 마이그레이션

Flyway는 데이터베이스 구조 변경을 파일로 관리하는 도구입니다.

## 왜 필요한가

개발 중에는 테이블이 계속 바뀝니다.

- `races` 테이블 추가
- `users` 테이블 추가
- `user_favorites` 테이블 추가
- 인덱스 추가
- 컬럼 타입 수정

이런 변경을 사람마다 직접 DB에 넣으면 환경이 쉽게 달라집니다. Flyway는 정해진 SQL 파일을 순서대로 실행해서 DB 상태를 맞춥니다.

## RacePulse의 마이그레이션 위치

```text
backend/src/main/resources/db/migration/
  V1__create_basic_race_tables.sql
  V2__create_auth_tables.sql
  V3__create_push_tables.sql
  V4__fix_enum_columns_to_varchar.sql
  V5__create_user_profile_tables.sql
```

## 파일 이름 규칙

```text
V번호__설명.sql
```

예시는 다음과 같습니다.

```text
V5__create_user_profile_tables.sql
```

중요한 규칙은 다음과 같습니다.

- `V` 뒤 숫자는 실행 순서입니다.
- 언더스코어 두 개 `__` 뒤에는 사람이 읽는 설명을 씁니다.
- 이미 실행된 파일은 수정하지 않는 것이 원칙입니다.
- 변경이 필요하면 새 버전 파일을 추가합니다.

## Flyway가 실행되는 방식

Spring Boot가 시작될 때 Flyway는 DB 안의 `flyway_schema_history` 테이블을 확인합니다.

```text
1. 현재 DB에 적용된 마이그레이션 목록 확인
2. 아직 실행되지 않은 V파일 찾기
3. 버전 순서대로 SQL 실행
4. 성공하면 flyway_schema_history에 기록
```

## `ddl-auto=validate`와의 관계

JPA에는 테이블을 자동 생성하거나 검증하는 설정이 있습니다.

RacePulse는 개발 환경에서 Flyway가 테이블을 만들고, JPA는 엔티티와 DB 구조가 맞는지 검증하는 방향이 좋습니다.

```text
Flyway: 테이블 생성/변경 담당
JPA validate: Java 엔티티와 DB 구조가 맞는지 확인
```

이렇게 나누면 운영 DB에서 JPA가 실수로 테이블을 바꾸는 위험을 줄일 수 있습니다.

## 마이그레이션 작성 예시

```sql
create table if not exists user_favorites (
    id bigint generated by default as identity primary key,
    user_id uuid not null references users(id) on delete cascade,
    target_type varchar(20) not null,
    target_id bigint not null,
    created_at timestamp not null default current_timestamp,
    unique (user_id, target_type, target_id)
);
```

이 SQL은 사용자의 즐겨찾기 정보를 저장합니다.

- `user_id`: 어떤 사용자의 즐겨찾기인지 표시
- `target_type`: 말, 경주, 기수 등 즐겨찾기 대상 종류
- `target_id`: 대상의 실제 ID
- `unique`: 같은 사용자가 같은 대상을 중복 저장하지 못하게 막음
- `on delete cascade`: 사용자가 삭제되면 해당 즐겨찾기도 함께 삭제

## 인덱스가 필요한 이유

인덱스는 DB 조회 속도를 높이는 장치입니다.

예를 들어 사용자의 즐겨찾기를 자주 조회한다면 `user_id`에 인덱스가 있으면 좋습니다.

```sql
create index if not exists idx_user_favorites_user
on user_favorites (user_id);
```

## JSONB를 쓰는 이유

PostgreSQL의 `jsonb`는 JSON 데이터를 저장하면서도 검색과 인덱싱에 유리한 타입입니다.

RacePulse에서는 경마장 트랙 종류처럼 배열 형태 데이터에 사용할 수 있습니다.

```sql
track_types jsonb not null default '[]'::jsonb
```

## 주의해야 할 실수

### 이미 적용된 V파일 수정

이미 실행된 마이그레이션 파일을 수정하면 다른 개발자 DB나 운영 DB에서 체크섬 오류가 날 수 있습니다. 수정 대신 새 파일을 추가합니다.

### Java enum과 PostgreSQL enum 불일치

Java 코드의 enum 값과 DB enum 값이 다르면 저장/조회 오류가 납니다. 이 문제를 줄이려면 운영 정책에 따라 `varchar`를 쓰거나, enum 변경 절차를 명확히 둬야 합니다.

### ML 서버에서 런타임 테이블 생성

현재 몬테카를로 결과 테이블은 ML 서버 코드가 없으면 생성하는 구조입니다. 편의성은 있지만, 운영 안정성을 위해 장기적으로는 Flyway 파일로 옮기는 것이 좋습니다.

## 새 테이블을 추가할 때 체크리스트

1. 테이블 이름이 프로젝트 네이밍과 맞는지 확인합니다.
2. 기본키와 외래키를 정합니다.
3. 중복 방지 제약이 필요한지 확인합니다.
4. 자주 조회하는 컬럼에 인덱스가 필요한지 확인합니다.
5. 한글 저장 컬럼은 `varchar`, `text`, `jsonb`가 UTF-8 환경에서 안전한지 확인합니다.
6. 기존 실행 파일을 수정하지 말고 새 `V번호__설명.sql` 파일을 만듭니다.
