# 04. APScheduler와 Redis

RacePulse는 외부 데이터를 자동으로 가져와야 합니다. 사람이 매번 버튼을 누를 수 없으므로 스케줄러가 필요합니다.

## APScheduler란

APScheduler는 Python에서 정해진 시간에 함수를 실행해주는 도구입니다.

예를 들어 다음과 같은 일을 자동화할 수 있습니다.

- 매일 아침 날씨 수집
- 매주 특정 요일 출전표 수집
- 경주가 끝난 뒤 결과 수집
- AI 해설 자동 생성

## RacePulse의 구현 위치

```text
ml-server/app/scheduler/scheduler.py
ml-server/app/core/redis_client.py
```

## Cron 표현식

Cron은 시간을 규칙으로 표현하는 방식입니다.

RacePulse 코드에서는 다음처럼 사용합니다.

```python
s.add_job(
    self.collect_weather_short,
    "cron",
    hour=6,
    minute=30,
    id="weather_short_morning",
    replace_existing=True,
)
```

뜻은 매일 06:30에 `collect_weather_short` 함수를 실행하라는 의미입니다.

## 스케줄 작업 예시

| 작업 | 실행 시점 | 목적 |
| --- | --- | --- |
| 주말 경주 결과 수집 | 월요일 14:00 | 토/일 경주 결과 저장 |
| 월요일 경주 결과 수집 | 화요일 09:00 | 월요일 경주 결과 저장 |
| 출전표 초안 수집 | 목요일 10:00 | 주말 경주 출전 정보 확보 |
| 출전표 변경 확인 | 목요일 17:30 | 변경사항 반영 |
| 단기 날씨 수집 | 매일 06:30, 18:30 | 경마장 날씨 업데이트 |
| 중기 날씨 수집 | 매일 06:00 | 3~10일 예보 업데이트 |
| API 호출 카운터 초기화 | 매일 00:00 | 외부 API 호출 제한 초기화 |

## Redis가 필요한 이유

스케줄러만 있으면 문제가 생길 수 있습니다.

예를 들어 서버가 두 대 떠 있으면 같은 작업이 동시에 두 번 실행될 수 있습니다. 또는 외부 API 호출 제한을 초과할 수 있습니다.

Redis는 이런 문제를 막는 빠른 임시 저장소로 사용됩니다.

## 분산 락

분산 락은 여러 서버 중 하나만 작업을 실행하도록 막는 장치입니다.

```text
1. 작업 시작 전 Redis에 락을 잡음
2. 락을 잡은 서버만 수집 실행
3. 작업이 끝나면 락 해제
4. 락을 못 잡은 서버는 작업을 건너뜀
```

이 방식은 같은 데이터가 중복 저장되는 문제를 줄여줍니다.

## TTL이 중요한 이유

TTL은 Redis 키가 자동으로 사라지는 시간입니다.

락을 잡은 서버가 중간에 죽으면 락이 영원히 남을 수 있습니다. TTL을 설정하면 일정 시간이 지난 뒤 락이 자동 해제됩니다.

```text
락 TTL = 작업 예상 최대 시간보다 조금 길게 설정
```

## Rate Limit 관리

외부 API는 하루 호출 횟수 제한이 있는 경우가 많습니다. Redis 카운터를 쓰면 호출 횟수를 빠르게 기록할 수 있습니다.

```text
1. API 호출 전 오늘 호출 횟수 확인
2. 제한에 도달했으면 호출 중단
3. 호출 가능하면 API 요청 후 카운터 증가
4. 매일 00:00에 카운터 초기화
```

## 체크포인트

체크포인트는 마지막으로 작업이 성공한 시점을 저장하는 값입니다.

예를 들어 수집 작업이 실패했을 때 체크포인트를 보면 언제까지 정상 처리됐는지 알 수 있습니다.

## 장애 상황에서 확인할 것

| 증상 | 확인할 위치 |
| --- | --- |
| 수집이 두 번 실행됨 | Redis 락 키, 서버 인스턴스 수 |
| 수집이 아예 안 됨 | APScheduler 실행 로그, 작업 등록 여부 |
| API 제한 초과 | Redis 호출 카운터 |
| 특정 작업만 실패 | 해당 작업 로그와 외부 API 응답 |
| 서버 재시작 후 작업 누락 | scheduler 시작 함수가 FastAPI 시작 시 호출되는지 확인 |

## 초보자가 기억할 핵심

APScheduler는 "언제 실행할지"를 담당합니다. Redis는 "동시에 실행해도 안전한지"와 "얼마나 호출했는지"를 관리합니다.
