# 00. 전체 기술 지도

RacePulse는 경마 데이터를 수집하고, 예측하고, 사용자가 보기 좋은 화면으로 보여주는 서비스입니다. 기술을 한 줄로 정리하면 다음과 같습니다.

> 프론트엔드가 Spring Boot API를 호출하고, Spring Boot는 PostgreSQL과 Redis를 사용하며, 예측/수집/시뮬레이션은 FastAPI 기반 ML 서버가 담당합니다.

## 전체 흐름

```text
사용자 브라우저
  ↓
React + Vite + PWA
  ↓
Spring Boot API
  ↓                 ↘
PostgreSQL           FastAPI ML 서버
  ↑                 ↓
Flyway          KRA API / 날씨 API / ML 모델 / 몬테카를로
  ↑
Redis: 캐시, 락, 호출 제한
```

## 각 기술이 필요한 이유

### React

사용자가 보는 화면을 만드는 기술입니다. RacePulse에서는 홈, 경주 목록, 예측 페이지, 검색, 마이페이지 같은 화면을 구성합니다.

### Vite

프론트엔드 개발 서버와 빌드 도구입니다. 개발 중에는 빠르게 화면을 띄우고, 배포 전에는 브라우저가 읽기 좋은 정적 파일로 묶어줍니다.

### React Query

API 데이터를 가져오고 캐시하는 도구입니다. 예를 들어 경주 목록을 여러 화면에서 다시 요청할 때 불필요한 중복 호출을 줄여줍니다.

### Spring Boot

백엔드 API 서버입니다. 프론트엔드가 직접 DB에 접근하지 않도록 중간에서 안전하게 데이터를 제공합니다.

### Spring Security

로그인 사용자와 비로그인 사용자를 구분하고, 보호된 API를 막아주는 보안 계층입니다.

### JWT

로그인 성공 후 클라이언트가 들고 다니는 인증 표식입니다. 사용자는 매 요청마다 JWT를 보내고, 백엔드는 이 토큰이 유효한지 확인합니다.

### Kakao OAuth

카카오 계정으로 로그인하는 기능입니다. RacePulse는 사용자의 카카오 비밀번호를 직접 다루지 않고, 카카오에서 검증된 로그인 결과만 받습니다.

### PostgreSQL

핵심 데이터베이스입니다. 경주, 경주마, 사용자, 즐겨찾기, 예측 결과 등을 저장합니다.

### Flyway

데이터베이스 구조 변경 이력을 관리합니다. `V1__...sql`, `V2__...sql`처럼 순서가 있는 SQL 파일을 실행해서 DB 구조를 일관되게 맞춥니다.

### Redis

빠른 임시 저장소입니다. RacePulse에서는 중복 수집 방지용 락, 외부 API 호출 횟수 제한, 캐시 같은 용도로 사용합니다.

### FastAPI

Python 기반 ML 서버입니다. 데이터 수집, 예측, 몬테카를로 시뮬레이션처럼 Python 생태계가 강한 작업을 맡습니다.

### APScheduler

정해진 시간에 자동으로 작업을 실행합니다. 예를 들어 매일 아침 날씨를 수집하거나, 특정 요일에 출전표를 수집합니다.

### 몬테카를로 시뮬레이션

확률을 여러 번 무작위로 뽑아 결과 분포를 계산하는 방법입니다. RacePulse에서는 각 말의 승률 예측값을 기반으로 1위, 2위, 3위 가능성을 여러 번 실험합니다.

### Web Push와 VAPID

브라우저 푸시 알림 기능입니다. VAPID는 서버가 합법적인 발신자임을 증명하는 키 체계입니다.

## 처음 배울 때 추천 순서

1. 화면이 어떻게 API를 부르는지 먼저 이해합니다.
2. API가 DB에서 데이터를 가져오는 흐름을 봅니다.
3. Flyway로 DB 구조가 어떻게 만들어지는지 확인합니다.
4. ML 서버가 예측 데이터를 어떻게 만들고 저장하는지 봅니다.
5. Redis와 APScheduler로 자동 수집 안정성을 이해합니다.
6. 인증, 푸시, PWA 같은 사용자 경험 기능을 붙여서 이해합니다.
