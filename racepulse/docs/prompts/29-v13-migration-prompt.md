## 구현 사항 (V13 SQL 전체)

### 1. `trainer_changes` — 조교사 변경 이력

조교사가 바뀐 경우 이력을 저장합니다.
변경 종류 뱃지: 🟠 주황색 (영향도: 중간)

```sql
CREATE TABLE IF NOT EXISTS trainer_changes (
    id              BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    horse_id        BIGINT       NOT NULL REFERENCES horses(id),
    race_id         BIGINT       REFERENCES races(id),
    old_trainer_id  BIGINT       REFERENCES trainers(id),
    new_trainer_id  BIGINT       NOT NULL REFERENCES trainers(id),
    detected_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    change_date     DATE         NOT NULL,
    -- previous_change_id: A→B→A 체인 처리 (14차 회의 확정사항)
    -- 이전 변경 기록의 id를 가리켜 변경 흐름을 추적합니다.
    previous_change_id BIGINT    REFERENCES trainer_changes(id)
);
```

인덱스: `idx_trainer_changes_horse`, `idx_trainer_changes_race`

### 2. `equipment_changes` — 장비 변경 이력 (블링커 등)

장비(블링커, 안장, 편자 등)가 바뀐 경우 이력을 저장합니다.
변경 종류 뱃지: 🟡 노랑 (영향도: 중간)

```sql
CREATE TABLE IF NOT EXISTS equipment_changes (
    id              BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    horse_id        BIGINT       NOT NULL REFERENCES horses(id),
    race_id         BIGINT       REFERENCES races(id),
    -- 장비 종류: BLINKER(블링커), SADDLE(안장), SHOE(편자) 등
    equipment_type  VARCHAR(20)  NOT NULL,
    old_value       VARCHAR(50),
    new_value       VARCHAR(50)  NOT NULL,
    detected_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    change_date     DATE         NOT NULL,
    -- blinker_first_time: 블링커를 처음 착용하는지 여부 (ML 피처: 14차 회의 확정)
    blinker_first_time BOOLEAN   NOT NULL DEFAULT FALSE,
    previous_change_id BIGINT    REFERENCES equipment_changes(id)
);
```

인덱스: `idx_equipment_changes_horse`, `idx_equipment_changes_race`

### 3. `user_wallets` — 편자 지갑 (Freemium 화폐 시스템)

14차 회의 확정: 편자 이원화 전략
- 🔩 이벤트 편자 (은색): 출석·광고·퀴즈로 획득, 6개월 만료, 소비 1순위
- 🥇 구매 편자 (금색): 인앱결제로 획득, 무기한, 소비 2순위
- 🌾 건초: 다마고치 전용 화폐 (Phase 5에서 사용)

```sql
CREATE TABLE IF NOT EXISTS user_wallets (
    id                  BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    -- users 테이블과 1:1 연결 (한 유저 = 지갑 하나)
    user_id             UUID         NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    -- 🔩 이벤트 편자 (은색): 출석체크·광고·퀴즈로 무료 획득, 6개월 뒤 만료
    event_horseshoe     INTEGER      NOT NULL DEFAULT 0 CHECK (event_horseshoe >= 0),
    event_horseshoe_expires_at TIMESTAMPTZ,
    -- 🥇 구매 편자 (금색): 인앱결제로만 획득, 만료 없음
    paid_horseshoe      INTEGER      NOT NULL DEFAULT 0 CHECK (paid_horseshoe >= 0),
    -- 🌾 건초: 다마고치 미니게임 전용 화폐 (편자와 교환 불가)
    hay                 INTEGER      NOT NULL DEFAULT 0 CHECK (hay >= 0),
    created_at          TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

인덱스: `idx_user_wallets_user`

### 4. `wallet_transactions` — 편자 거래 내역

편자가 언제, 왜 얼마나 증감했는지 추적합니다.
(결제 환불, 소비 취소, 관리자 지급 등 감사 로그 역할)

```sql
CREATE TABLE IF NOT EXISTS wallet_transactions (
    id              BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id         UUID         NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    -- 거래 종류: EARN_ATTENDANCE(출석), EARN_AD(광고), EARN_QUIZ(퀴즈),
    --            EARN_PURCHASE(결제), SPEND_TOP1, SPEND_ENSEMBLE, SPEND_AI_PRE,
    --            SPEND_COUNTERFACTUAL, ADMIN_GRANT, ADMIN_REVOKE, EXPIRE
    transaction_type VARCHAR(30) NOT NULL,
    -- 화폐 종류: EVENT(이벤트편자), PAID(구매편자), HAY(건초)
    currency_type   VARCHAR(10)  NOT NULL,
    -- 변화량: 양수=획득, 음수=소비
    amount          INTEGER      NOT NULL,
    -- 거래 후 잔액 (이벤트/구매 구분)
    balance_after   INTEGER      NOT NULL,
    description     TEXT,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

인덱스: `idx_wallet_tx_user`, `idx_wallet_tx_type`, `idx_wallet_tx_created`

### 5. `users` 테이블 컬럼 추가 — 개인정보보호법 동의

14차 회의 확정: 별도 팝업 방식, 스크롤 완료 후 "확인함" → "동의하기" 2단계

```sql
-- 마케팅 정보 수신 동의 여부 (선택)
ALTER TABLE users ADD COLUMN IF NOT EXISTS
    marketing_agreed    BOOLEAN  NOT NULL DEFAULT FALSE;

-- 이용약관 동의 시각 (필수, 가입 시 NULL이면 동의 안 한 것)
ALTER TABLE users ADD COLUMN IF NOT EXISTS
    terms_agreed_at     TIMESTAMPTZ;

-- 동의한 약관 버전 (버전 업데이트 시 재동의 여부 판단에 사용)
ALTER TABLE users ADD COLUMN IF NOT EXISTS
    terms_version       VARCHAR(10);
```

### 6. `ai_commentary` 테이블 컬럼 추가 — AI 해설 품질 관리

14차 회의 확정: 품질 점수 0~100점, HIGH(80+)/MED(50~79)/LOW(-50), 연속 LOW 3회 시 관리자 알림

```sql
-- AI 해설 품질 점수 (0~100): 사행성 필터·완성도 등을 종합 채점
ALTER TABLE ai_commentary ADD COLUMN IF NOT EXISTS
    quality_score       SMALLINT    CHECK (quality_score BETWEEN 0 AND 100);

-- 이 해설 생성에 사용된 GPT temperature 값 (사전해설: 0.7, 결과해설: 0.3)
ALTER TABLE ai_commentary ADD COLUMN IF NOT EXISTS
    temperature_used    NUMERIC(3, 2);

-- 사행성 필터 걸려서 재생성한 횟수 (최대 2회 retry 후 fallback 템플릿 사용)
ALTER TABLE ai_commentary ADD COLUMN IF NOT EXISTS
    retry_count         SMALLINT    NOT NULL DEFAULT 0;

-- GPT API 응답 시간 (밀리초): 성능 모니터링용
ALTER TABLE ai_commentary ADD COLUMN IF NOT EXISTS
    generation_ms       INTEGER;
```

## 주석 요구사항 ⚠️ 반드시 지켜주세요

이 프로젝트의 개발자는 **코딩 입문자**입니다.
**학생을 가르친다는 생각**으로 모든 SQL에 한국어 주석을 달아주세요.

- `GENERATED BY DEFAULT AS IDENTITY`가 무엇인지 설명 (자동 증가 ID)
- `REFERENCES`(외래키)가 왜 필요한지, ON DELETE CASCADE가 뭔지 설명
- `CHECK (amount >= 0)`같은 제약조건이 왜 필요한지 설명
- `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`가 왜 IF NOT EXISTS인지 설명
- `TIMESTAMPTZ` vs `TIMESTAMP` 차이 설명 (타임존 포함 여부)
- 인덱스가 왜 필요한지, 어떤 쿼리에서 빨라지는지 설명

## ⚠️ 주의사항

1. **V12 이하 파일 절대 수정 금지** — Flyway 체크섬 오류 발생
2. `IF NOT EXISTS` / `IF EXISTS` 사용 필수 — 재실행 안전성 보장
3. `trainer_changes`의 `previous_change_id`는 자기참조(self-referencing FK) — 같은 테이블을 가리킴
4. `user_wallets`는 users와 1:1 관계 — UNIQUE 제약 필수
5. `wallet_transactions`는 삭제 없이 쌓기만 함 — DELETE 쿼리 금지
